# MCP Tools Delta: Campaign Editor

These tools extend the feature-001 MCP server. They must reuse the same
OpenAPI-derived validation, authorization, and audit behavior as the HTTP API.

## `campaign_editor_get_draft`

**Purpose**: Read the campaign editor draft available to the caller.

**Input**:
- `campaignId` string, required

**Output**:
- `campaignId`
- `topic`
- `templateContent`
- `placeholderNames`
- `status`
- `updatedAt`

**Authorization**: Admins can read any campaign. Operators can read only assigned
campaigns.

## `campaign_editor_save_draft`

**Purpose**: Save sanitized campaign editor content generated by supported
toolbar actions.

**Input**:
- `campaignId` string, required
- `topic` string, required
- `templateContent` string, required

**Output**:
- Saved campaign editor draft
- Sanitization warnings
- Detected placeholder names

**Validation**:
- Direct HTML authoring is not trusted.
- Backend sanitization must enforce the same allowlist as the browser.
- Placeholder syntax must use `{{ value }}`.

## `campaign_editor_validate`

**Purpose**: Validate campaign placeholders against recipient variables before
send.

**Input**:
- `campaignId` string, required

**Output**:
- `canSend`
- Detected placeholders
- Per-recipient missing variables and fallback usage

## `campaign_send_outcomes`

**Purpose**: List per-recipient send outcomes for campaign recovery review.

**Input**:
- `campaignId` string, required

**Output**:
- Per-recipient `pending`, `submitted`, `failed`, `uncertain`, or
  `force_resend_queued` states
- Whether each recipient requires review
- Whether force resend is allowed

## `campaign_force_resend_uncertain_recipient`

**Purpose**: Queue a resend for an uncertain recipient after the caller accepts
duplicate risk.

**Input**:
- `campaignId` string, required
- `contactId` string, required
- `acknowledgedDuplicateRisk` boolean, required and must be `true`
- `reason` string, optional

**Output**:
- `campaignId`
- `contactId`
- `sendJobId`
- `status: force_resend_queued`

**Authorization and Safety**:
- Only admins or the assigned operator can force resend.
- The recipient must currently be uncertain.
- The tool enqueues asynchronous work and must not call EmailLabs inline.
- The acknowledgement is audited.
